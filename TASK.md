# Тестовое задание: Backend Engineer (Go)

## Название: Wishlist API

### 1. Цель задания

Реализовать REST API сервис, в котором пользователь может зарегистрироваться, создать вишлист к празднику или событию, наполнить его подарками и открыть доступ по уникальной ссылке.

Результат — ссылка на публичный Git-репозиторий.

---

### 2. Технический стек

* Git
* Docker Compose
* Go 1.21+ (набор библиотек на ваше усмотрение)
* PostgreSQL
* Миграции для управления схемой БД (например, goose, golang-migrate)

---

### 3. Функциональные требования

#### 3.1. Авторизация
- Регистрация и вход по email + пароль.
- Пароли хранятся хэшированно.
- Закрытые эндпоинты доступны только авторизованным пользователям.

#### 3.2. Вишлисты (CRUD)
- Авторизованный пользователь создаёт, просматривает, редактирует и удаляет свои вишлисты.
- У вишлиста есть название события, описание, дата.

#### 3.3. Позиции в вишлисте (CRUD)
- Внутри вишлиста список позиций (подарков).
- У позиции: название, описание, ссылка на товар, приоритет (степень желания подарка).

#### 3.4. Публичный доступ по ссылке
- При создании вишлиста генерируется уникальный токен.
- Необходимо реализовать GET эндпоинт, который по этому токену возвращает вишлист со всеми позициями без авторизации.
- Необходимо реализовать эндпоинт бронирования подарка: принимает токен вишлиста и ID позиции, помечает позицию как забронированную. Авторизация не требуется. Если позиция уже забронирована — возвращать ошибку.

---

### 4. Нефункциональные требования

#### 4.1. Запуск
Проект запускается командой:
```bash
docker-compose up --build
```
Никаких дополнительных шагов не требуется.

#### 4.2. Конфигурация
Все параметры вынесены в переменные окружения. В репозитории есть `.env.example`.

#### 4.3. API
REST, ответы в JSON. Входные данные валидируются, ошибки возвращаются с корректными HTTP-кодами.

#### 4.4. README
В корне проекта файл `README.md`: что делает сервис, как запустить, примеры запросов.

#### 4.5. Git
Репозиторий публичный, без секретов и ключей в коде, с осмысленной историей коммитов.

---
